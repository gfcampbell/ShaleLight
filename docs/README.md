# ShaleLight Documentation

This folder contains implementation-level documentation for setup, architecture, rationale, and operations.

## Documents

- `../setup-wizard.html` - Canonical setup wizard entrypoint at repo root.
- `setup-wizard.html` - Offline wizard implementation file loaded by the root entrypoint.
- `ARCHITECTURE.md` - System layout and component boundaries.
- `RAG_PIPELINE.md` - Crawl-to-answer pipeline behavior.
- `SEARCH_STRATEGY.md` - Retrieval strategy and scoring rationale.
- `SECURITY_MODEL.md` - Security assumptions, controls, and boundaries.
- `OPERATIONS.md` - Runbook for setup, migrations, backups, and maintenance.
- `TROUBLESHOOTING.md` - Common failures and fixes.
- `DECISIONS.md` - Key technical decisions and the rationale behind them.

## Audience

- Operators deploying ShaleLight in local or air-gapped environments.
- Engineers reviewing implementation details and tradeoffs.
- Contributors who need context before changing core behavior.
